# Pulse HR — Marketing

Static, SEO-first Astro site for `pulsehr.com`. The main React SPA lives at `../` and is served from `app.pulsehr.com` (or `/app/*`); this app owns marketing surfaces only.

## Run

```bash
cd marketing
bun install
bun run dev      # http://localhost:4321
bun run build    # → dist/
bun run preview  # serve dist/ locally
```

## What's implemented (SEO audit closure)

- **Prerender** — Astro ships static HTML. Googlebot, OAI-SearchBot, PerplexityBot, ClaudeBot all see full content.
- **Root at `/`** — marketing copy is at the apex. App links point to `app.pulsehr.com`.
- **`public/robots.txt`** — allows AI crawlers explicitly, points to sitemap.
- **Sitemap** — auto-generated by `@astrojs/sitemap` at `/sitemap-index.xml`.
- **Canonical** — `<link rel="canonical">` per route via `BaseLayout.astro`.
- **OG + Twitter** — per-route tags, 1200×630 OG image reference at `/og/landing.png` (drop asset in `public/og/`).
- **JSON-LD** — `Organization` (every page), `SoftwareApplication` + `FAQPage` (landing only).
- **H1 keywords** — hero eyebrow: "HR, payroll & time tracking in one workspace"; retained brand-voice H1 beneath.
- **Commessa gloss** — first use in hero disambiguates for English readers.
- **Internal links** — footer columns and Changelog/Docs/Pricing CTAs point to real slugs (pages to be scaffolded as content grows).
- **Accessibility** — skip-to-main link, `<h1>`→`<h2>` hierarchy, `aria-labelledby` on every section, `aria-label` on decorative SVGs, `<details>`/`<summary>` for FAQ (no JS required).
- **Performance** — static HTML + one tiny vanilla `<script>` in `ProductPreview`. No React hydration. Google fonts preconnect + `display=swap`.

## Architecture

```
marketing/
├── astro.config.mjs        # Astro + sitemap + tailwind/vite
├── public/
│   ├── robots.txt
│   └── favicon.svg
└── src/
    ├── data/landing.ts     # All content arrays (FEATURES, LABS, FAQ…)
    ├── layouts/BaseLayout.astro  # SEO meta + JSON-LD + skip-link
    ├── components/         # 15 section components
    ├── pages/index.astro   # Wires sections
    └── styles/global.css   # Tailwind 4 theme + utilities
```

Every content change happens in `src/data/landing.ts`. Sections are presentational; they import and map.

## Deploy

Point Vercel (or any static host) at `marketing/` with:
- Build: `bun run build`
- Output: `dist/`
- Framework: Astro (auto-detected)

DNS: `pulsehr.com` → this app. `app.pulsehr.com` → the React SPA in `../`.
